## 2. 测试策略

### 2.1 Rust 单元测试（纯逻辑，优先覆盖）

- `project_slug()`：用 [04 §2.3](./04-数据模型与注入.md) 的验证表做断言（`G:\hty_workflows`→`G--hty-workflows` 等）。
- `parse_frontmatter()`：正常 / 缺字段 / 无 frontmatter / 非法 YAML 四类用例。
- `inject::build_text()`：profile × itemType 的全矩阵断言（claude-skill→`/x`，codex→`@path`…）。
- skill `invoke` 推导：user/project/plugin 三类来源。

### 2.2 集成 / 手动 e2e（关键路径）

| 场景 | 验证点 |
|---|---|
| PTY 回显 | 启动 shell，`echo hello` 能回显；中文、退格、Ctrl+C 正常 |
| Claude 启动 | `launch_cmd` 起 claude，TUI 渲染正常、可交互 |
| 多终端隔离 | 两终端同时刷输出，互不串扰 |
| Resize | 反复缩放窗口/分屏，claude 排版不乱 |
| 拖拽注入 | skill→`/name`、memory→`@path` 被对应 CLI 正确识别 |
| 实时刷新 | 新建/改/删 skill 或 memory 文件，列表自动更新 |
| 进程清理 | 关面板/关应用后无残留 claude/shell 进程（任务管理器核对） |
| 布局恢复 | 重启后三栏宽度与 dock 结构恢复 |

### 2.3 前端

- 组件层：`SkillPanel`/`MemoryPanel` 列表渲染与搜索过滤（fuse）。
- 拖放：`registerDropTarget` 的 over/leave/drop 行为（jsdom 模拟 dataTransfer）。
- IPC 封装层用 mock `invoke` 做契约测试。

## 3. 打包与分发

### 3.1 构建

```powershell
pnpm tauri build
```

产物默认在 `src-tauri/target/release/bundle/`：
- **NSIS**（`.exe` 安装器，推荐，体积小、可自定义）
- **MSI**（`.msi`，适合企业/组策略部署）

在 `tauri.conf.json` 的 `bundle.windows` 配置 NSIS/WiX 选项、图标、产品名（HtyBox）、版本号。

### 3.2 运行时依赖

- 目标机需 **WebView2 Runtime**（Win10/11 多数自带）。Tauri 可配置 `webviewInstallMode` 在缺失时引导安装或随包携带（bootstrapper/offline）。
- 无需随包带 Node：前端是构建后的静态资源，由 WebView 加载。

### 3.3 签名与更新（可选，后续）

- **代码签名**：避免 SmartScreen 警告，需购买证书后在 bundler 配置签名。
- **自动更新**：接 `tauri-plugin-updater` + 版本清单托管（GitHub Releases 等）。v1 可先手动分发。

## 4. 开发环境搭建（实现者清单）

1. 安装 **Rust**（rustup）、**Node 20+**、**pnpm**。
2. 安装 **Visual Studio Build Tools**，勾选"使用 C++ 的桌面开发"（编译 Rust/Tauri 原生依赖必需）。
3. 确认 **WebView2 Runtime** 已安装。
4. `pnpm create tauri-app@latest` → 选 React + TypeScript + Vite。
5. `pnpm install` → `pnpm tauri dev` 验证窗口可起。
6. 按 [05 里程碑](./05-里程碑与任务拆解.md) 从 M0 推进。

## 5. 上线后可演进方向（v2+）

- memory 作用域**跟随聚焦终端的 cwd** 自动切换。
- skill/memory **就地编辑器**（不止浏览，可改 frontmatter/正文）。
- **拖拽预览**：拖动时显示将注入的确切文本气泡。
- **会话恢复**：记录每终端最近命令/cwd，重启智能重建。
- 支持 **SSH/远程终端**、多窗口、跨平台（macOS/Linux —— portable-pty 与 Tauri 均跨平台，主要是 Profile 与路径适配）。
- 与 Claude Code 的 `/mcp`、自定义 agent 等进一步联动。
